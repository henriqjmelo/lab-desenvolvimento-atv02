# Historias de Usuario - Sistema de Aluguel de Carros

## Objetivo

Documentar as principais necessidades funcionais do sistema sob a perspectiva dos atores cliente e agente.

## Historias (Cliente e Agente)

| ID | Historia |
| -- | -- |
| 01 | Eu, como cliente, quero me cadastrar no sistema para poder acessar a plataforma. |
| 02 | Eu, como cliente, quero fazer login para utilizar as funcionalidades do sistema. |
| 03 | Eu, como cliente, quero criar um pedido de aluguel de carro para solicitar um veiculo. |
| 04 | Eu, como cliente, quero consultar o status do meu pedido para acompanhar a solicitacao. |
| 05 | Eu, como cliente, quero modificar ou cancelar meu pedido para ter flexibilidade antes da aprovacao. |
| 06 | Eu, como agente, quero avaliar financeiramente os pedidos para decidir sua aprovacao. |
| 07 | Eu, como agente, quero modificar pedidos quando necessario para ajustar informacoes. |
| 08 | Eu, como cliente, quero formalizar o contrato apos aprovacao para concluir o aluguel. |

## Observacoes

- As historias 01 a 07 possuem cobertura funcional no prototipo atual.
- A historia 08 esta modelada e parcialmente preparada no dominio (ContratoCredito), podendo ser expandida em UI/fluxo completo.
